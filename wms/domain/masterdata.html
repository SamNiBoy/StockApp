Master Data

WMS system running with certain master data:

  • Item
  • Item Footprint
  • Supplier
  • Address
  • Facility

Item: Item is an ID identifier used in WMS for a specific item, alternatively it is called SKU, or Part Number in JDA WMS.

Item Footprint: Footprint defines package information for an item, all named TiHi, it is defined by UOM(Unit Of Measure), with footprint we can define:

  • How many different UOMs for this item?
    • Most of item has at least 3 different UOM: EA, CS, and PA, some item has additional UOM like IP, LA etc. UOM defines an unique measure unit information: quantity, length, width, height, weight.
  • EA
    • It defines information for a single piece of the item, unit quantity is 1 which means 1EA = 1 piece, also it defines the length, height, and width dimension of 1 EA, and weigth for 1EA.
  • IP
    • It defines information for a single Inner Pack of the item, unit quantity is 5 which means 1IP = 5EA, also it defines the length, height, and width dimension of 1 IP, and weigth for 1IP.
  • CS
    • It defines information for a single case of the item, if one case has 2 Inner Pack then means 1CS = 2IP = 10EA, similary we can define the length, height, and width dimension of 1 case, and weight for 1CS.
  • LA
    • It defines information for a single layer of the item, if one layer has 10 case then means 1LA = 10CS = 20IP = 100EA, similary we can define the length, height, and width dimension of 1 layer, and weight for 1LA.
  • PA
    • It defines information for a single pallet of the item, if one pallet has 5 layers then means 1PA = 5LA = 50CS = 100IP = 500EA, similary we can define the length, height, and width dimension of 1 pallet, and weight for 1PA. PA is the biggest UOM for any item.

Supplier: Supplier defines who supplys a purchase order, with which the inbound shipment can be created for receiving. Supplier must be defined with an address information.

Address: Address defines some entity's address information, for example, Supplier has address, warehouse address, outbound order ship to address.

Facility: Facility defines the warehouse physical information:

  • Warehouse
    • Warehouse defines a brief information about a warehouse, like the name of the warehouse, address, timezone information.
  • Building
    • Building defines physical buildings defined in same warehouse.
  • Area
    • Area is a division for a physical area, in earlier version of JDA WMS, area plays an important role as there a number of configurations were defined in area level, however this has been changed as new design pulled these configurations into 3 different zones: Movement Zone, Pick Zone, and Storage Zone. Area now has less important role.
  • Location
    • Location is the most important master data which defines a single unit of space for storage, most importantly in location master data you can define, the location capacity code by whicy we tell system the location capacity is tracked by:
    • loccod = P
      • Pallet: Location capacity will be tracked by the number of Pallet. e.g if the maximum QVL is 5, it means in maximum this location can store 5 pallets by counting the number of LPN(License Pallet Number).
    • loccod = V
      • Volume: Location capacity will be tracked by the dimension of location height, width and length. e.g if location lenght, width and height are all 100CM, then the maximum QVL of the location will be 100*100*100 = 1000000CM^3. how many pallet or carton can be stored in the location depends on the dimension of pallet or carton which defined in item footprint UOM master data.
    • loccod = L
      • Length: Location capacity will be tracked by the dimension of location length. e.g if location lenght is 500CM, then the maximum QVL of the location will be 500CM. how many pallet or carton can be stored in the location depends on the length dimension of pallet, usually by the length of Handling Unit on which the pallet is sitting on.
    • loccod = E
      • Each: Location capacity will be tracked by a number of EA. e.g if location lenght is 500EA, then the maximum QVL of the location will be 500 eaches for any item.
    Which location code to use is determined by the business requirement to see which is fitting your business well. See Facility Setup for how to setup your warehouse.
