Trace Digger

Trace Digger was developed in 2010, at that time Sam worked in WMS maintainance team, from day to day we need to investigate a lot of traces for troubleshooting, there is already a very good tool available named LextEdit for investigating MOCA trace, however, a tool with more abilities are highly needed:

  • Ability for quick search and navigating
  • Ability for searching only withing selected command node
  • Ability for highlighting corresponding section when selecting a node
  • Ability for ordering command/SQL by performance
  • Ability for bookmarking a command/SQL for navigating between nodes
  • Ability for Investigating MTF trace
  • Ability for handling GB sized trace
  • Ability for spliting multi-threads trace by thread ID
  • Ability for customizing coloring schema with preferrence
  • ...

There is a document on the internet talked about more detail, See Here for more.

The latest version is 2.1, comparing to previous version, a few issues fixed:

  • Used a new font for hilighted trace
  • Made the search bar more wider, so can type more chars to search
  • Fixed the issue of can not open Unix system generated trace file
